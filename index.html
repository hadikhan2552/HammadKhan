⚠️ Getting "Permission Denied"?

This is a Firebase Database Rules issue — follow these steps to fix it:

Step 1 — Go to console.firebase.google.com
Step 2 — Open your project → click Realtime Database in the left sidebar
Step 3 — Click the Rules tab at the top
Step 4 — Delete everything in the rules box and paste this exactly:

{ "rules": { ".read": true, ".write": "auth != null", "hero": { ".validate": "newData.hasChildren(['first','last'])" }, "skills": { "$id": { ".validate": "newData.hasChildren(['title','desc'])" } }, "projects": { "$id": { ".validate": "newData.hasChildren(['title','desc'])" } }, "reviews": { "$id": { ".validate": "newData.hasChildren(['name','text','status'])" } } } }

These rules are secure: only you (logged-in admin) can write data. Anyone can read (needed to display your portfolio to visitors). The .validate rules prevent garbage/invalid data from being saved.


Step 5 — Click Publish
Step 6 — Come back here and try saving again ✅

These rules mean: anyone can READ your portfolio data (needed to show your site to visitors), but only YOU (logged-in admin) can WRITE/save data. This is exactly the correct secure setup.
